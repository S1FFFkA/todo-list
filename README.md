# Todo List API

HTTP-сервер на Go для управления списком задач (todo list) с REST API. Приложение использует in-memory хранилище и реализует полный набор CRUD операций для задач.

## Описание

Проект представляет собой веб-сервер, который позволяет создавать, получать, обновлять и удалять задачи. Каждая задача содержит заголовок, описание , статус выполнения и время создания/завершения задачи. Приложение использует только стандартную библиотеку Go и не требует внешних зависимостей для работы.





# Запуск проекта


Убедитесь, что установлен Go версии 1.25 или выше:



## Запуск через Docker

### Вариант 1: Запуск в фоновом режиме 

1. Соберите и запустите контейнер в фоновом режиме:
```bash
docker-compose up -d --build
```

После этой команды контейнер запустится в фоне, и вы сможете продолжать работать в консоли.

2. Просмотр логов:
```bash
docker-compose logs -f
```

3. Остановка контейнера:
```bash
docker-compose down
```

### Вариант 2: Запуск в foreground режиме

Если вы хотите видеть логи прямо в консоли:

```bash
docker-compose up --build
```


## Логирование

### Расположение логов

Логи выводятся в стандартный поток вывода (stdout) в формате JSON.  При запуске через Docker логи доступны через команду `docker-compose logs`.

### Формат логов

Логи структурированы в JSON формате и содержат следующие уровни:

- **INFO** - информационные сообщения (запуск сервера, обработка запросов)
- **WARN** - предупреждения (несуществующие задачи, ошибки валидации)
- **ERROR** - критические ошибки (внутренние ошибки сервера, ошибки декодирования)

## API Endpoints

### Создание задачи
```
POST /todos
Content-Type: application/json

{
  "headline": "Заголовок задачи",
  "description": "Описание задачи"
}
```

### Получение всех задач
```
GET /todos
```

### Получение задачи по ID
```
GET /todos/{id}
```

### Обновление задачи
```
PUT /todos/{id}

{
  "headline": "Новый заголовок",
  "description": "Новое описание",
}
```

### Завершение задачи
```
PATCH /todos/{id}
```

### Удаление задачи
```
DELETE /todos/{id}
```

## Особенности

- In-memory хранилище данных (данные не сохраняются между перезапусками)
- Генерация уникальных 8-значных ID для задач
- Структурированное логирование в JSON формате
- Graceful shutdown с таймаутом 5 секунд
- Полное покрытие тестами бизнес-логики
- Использование только стандартной библиотеки Go
- Добавил время создания и завершения задачи
## Технологии

- Go 1.25+
- Standard library only 
- Docker
